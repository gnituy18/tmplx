Introduction

Tmplx is a compile-time framework using Go for building state-driven web apps. It allows you to build UIs with React-like reactivity purely in Go.

Embed Go code in HTML to define states and event handlers. Go manages backend logic while HTML defines the UI, all in one file. This creates a seamless integration, eliminating the mental context switch between backend and frontend development.

Quick Start

1. Install tmplx

$ go install github.com/gnituy18/tmplx@latest

2. Create a new project

$ mkdir my-tmplx-app
$ cd my-tmplx-app
$ go mod init my-tmplx-app

$ mkdir pages
$ touch pages/index.html

$ mkdir tmplx   # generated code will go here

3. Write your first page (pages/index.html)

<!DOCTYPE html>
<html lang="en">
  <head>
    <title>My Tmplx App</title>

    <script type="text/tmplx">
      var name string = "World" // state
      var greeting string = fmt.Sprintf("Hello, %s!", name) // derived
    </script>
  </head>

  <body>
    <h1>{greeting}</h1>
  </body>
</html>

4. Generate routes

$ tmplx -out-file=tmplx/routes.go

This creates tmplx/routes.go with compiled handlers and render functions.

5. Add main.go

package main

import (
	"log"
	"net/http"

	"my-tmplx-app/tmplx"
)

func main() {
	for _, r := range tmplx.Routes() {
		http.HandleFunc(r.Pattern, r.Handler)
	}
	log.Fatal(http.ListenAndServe(":8080", nil))
}

6. Run the app

$ go run .

Open http://localhost:8080/. You should see “Hello, World!” rendered server-side using Go logic.
